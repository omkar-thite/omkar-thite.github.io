Writing Functions in Python

A comprehensive guide to writing effective, reusable, and well-documented Python functions.

1. Best Practices for Functions

Writing Good Docstrings

Docstrings explain what a function does, its parameters, and what it returns.

Google Style Docstrings

def function_name(param1, param2):
    """Brief description of function.
    
    Args:
        param1 (type): Description of param1.
        param2 (type): Description of param2.
    
    Returns:
        type: Description of return value.
    
    Raises:
        ErrorType: When this error occurs.
    
    Examples:
        >>> function_name(1, 2)
        3
    """
    return param1 + param2

NumPy Style Docstrings

def function_name(param1, param2):
    """
    Brief description of function.
    
    Parameters
    ----------
    param1 : type
        Description of param1.
    param2 : type
        Description of param2.
    
    Returns
    -------
    type
        Description of return value.
    
    Examples
    --------
    >>> function_name(1, 2)
    3
    """
    return param1 + param2

Accessing Docstrings

# Using __doc__ attribute
print(function_name.__doc__)

# Using inspect module
import inspect
print(inspect.getdoc(function_name))

2. Don't Repeat Yourself (DRY)

Avoid writing the same code multiple times. Extract repeated logic into functions.

Before (Repetitive)

# Same logic repeated
df1['new_col'] = df1['col1'] * 2 + df1['col2']
df2['new_col'] = df2['col1'] * 2 + df2['col2']
df3['new_col'] = df3['col1'] * 2 + df3['col2']

After (DRY)

def create_new_col(df):
    """Create new column from col1 and col2."""
    df['new_col'] = df['col1'] * 2 + df['col2']
    return df

df1 = create_new_col(df1)
df2 = create_new_col(df2)
df3 = create_new_col(df3)

3. Do One Thing

Each function should do one thing and do it well.

Bad Practice

def process_and_save_and_email(data):
    """Does too many things."""
    cleaned = clean_data(data)
    save_to_db(cleaned)
    send_email("Data processed")
    return cleaned

Good Practice

def clean_data(data):
    """Clean the data."""
    return cleaned_data

def save_data(data):
    """Save data to database."""
    pass

def notify_completion():
    """Send notification email."""
    pass

4. Context Managers

Context managers handle setup and teardown of resources automatically.

Using with Statement

# File handling with context manager
with open('file.txt', 'r') as f:
    content = f.read()
# File is automatically closed

Creating Custom Context Managers

Using a Class

class Timer:
    def __enter__(self):
        self.start = time.time()
        return self
    
    def __exit__(self, exc_type, exc_val, exc_tb):
        self.end = time.time()
        print(f"Elapsed: {self.end - self.start:.2f}s")
        return False  # Don't suppress exceptions

# Usage
with Timer():
    time.sleep(1)

Using contextlib.contextmanager

from contextlib import contextmanager

@contextmanager
def timer():
    """Context manager for timing code."""
    start = time.time()
    try:
        yield  # Code in 'with' block runs here
    finally:
        end = time.time()
        print(f"Elapsed: {end - start:.2f}s")

# Usage
with timer():
    time.sleep(1)

Context Manager with Value

@contextmanager
def database_connection(db_name):
    """Manage database connection."""
    conn = create_connection(db_name)
    try:
        yield conn  # Return connection to user
    finally:
        conn.close()

with database_connection('mydb') as conn:
    result = conn.execute('SELECT * FROM users')

5. Scope and Closures

Variable Scope

ScopeDescriptionKeyword
LocalInside the function-
EnclosingIn outer function (nested)nonlocal
GlobalAt module levelglobal
Built-inPython built-ins-

LEGB Rule

Python searches for variables in this order: Local → Enclosing → Global → Built-in

Using global

count = 0

def increment():
    global count
    count += 1

increment()
print(count)  # 1

Closures

A closure is a function that remembers values from its enclosing scope.

def make_multiplier(n):
    """Return a function that multiplies by n."""
    def multiplier(x):
        return x * n
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))  # 10
print(triple(5))  # 15

Using nonlocal

def counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

my_counter = counter()
print(my_counter())  # 1
print(my_counter())  # 2

6. Decorators

Decorators modify or enhance functions without changing their code.

Basic Decorator

def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("Before function")
        result = func(*args, **kwargs)
        print("After function")
        return result
    return wrapper

@my_decorator
def say_hello(name):
    print(f"Hello, {name}!")

say_hello("World")
# Before function
# Hello, World!
# After function

Preserving Function Metadata

from functools import wraps

def my_decorator(func):
    @wraps(func)  # Preserves __name__, __doc__, etc.
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

Decorator with Arguments

def repeat(n_times):
    """Decorator that repeats function n times."""
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n_times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def say_hi():
    print("Hi!")

say_hi()
# Hi!
# Hi!
# Hi!

Common Decorators

Timer Decorator

import time
from functools import wraps

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} took {end - start:.2f}s")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)
    
slow_function()  # slow_function took 1.00s

Memoization Decorator

def memoize(func):
    """Cache function results."""
    cache = {}
    @wraps(func)
    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]
    return wrapper

@memoize
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

print(fibonacci(100))  # Fast with memoization

Stacking Decorators

@decorator1
@decorator2
@decorator3
def my_function():
    pass

# Equivalent to:
# my_function = decorator1(decorator2(decorator3(my_function)))

7. Function Arguments

*args and **kwargs

def flexible_function(*args, **kwargs):
    """Accept any number of arguments."""
    print(f"args: {args}")
    print(f"kwargs: {kwargs}")

flexible_function(1, 2, 3, name="Alice", age=30)
# args: (1, 2, 3)
# kwargs: {'name': 'Alice', 'age': 30}

Default Mutable Arguments - Danger!

# BAD - mutable default argument
def append_to(element, to=[]):
    to.append(element)
    return to

print(append_to(1))  # [1]
print(append_to(2))  # [1, 2] - Unexpected!

# GOOD - use None as default
def append_to(element, to=None):
    if to is None:
        to = []
    to.append(element)
    return to

Best Practices Summary

  1. Write clear docstrings - Document parameters, returns, and examples
  2. DRY - Don't repeat yourself
  3. Single responsibility - Each function does one thing
  4. Use context managers - For resource management
  5. Use @wraps - Preserve function metadata in decorators
  6. Avoid mutable defaults - Use None instead
  7. Keep functions pure - Minimize side effects when possible
  8. Use closures - For encapsulating state
