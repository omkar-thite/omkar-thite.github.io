2. Date and Time Data Types

TIMESTAMP

  • Contains both date and time with microsecond precision
  • Uses ISO 8601 format: yyyy-mm-dd

INTERVAL

Stores time as a period (years, months, days, hours, seconds).

-- Calculate expected return date (3 days from rental)
SELECT rental_date + INTERVAL '3 days' AS expected_return
FROM rental;

3. Working with Arrays

Creating Tables with Arrays

CREATE TABLE grades (
    student_id INT,
    email TEXT[][],
    test_scores INT[][]
);

Accessing Arrays

⚠️ PostgreSQL arrays start at index 1, not 0!

SELECT 
    email[1][1] AS email_type,
    email[1][2] AS email_address,
    test_scores[1]
FROM grades;

Searching Arrays

-- Using ANY() Function
SELECT title, special_features 
FROM film 
WHERE 'Trailers' = ANY(special_features);

-- Using Contains Operator @>
SELECT title, special_features 
FROM film 
WHERE special_features @> ARRAY['Deleted Scenes'];

6. Extracting and Transforming Date/Time Data

EXTRACT() Function

SELECT EXTRACT(quarter FROM timestamp '2025-02-25 13:54:44') AS quarter;
SELECT EXTRACT(dow FROM rental_date) AS day_of_week FROM rental;

DATE_TRUNC() Function

SELECT DATE_TRUNC('year', TIMESTAMP '2025-02-25 13:54:44');
-- Result: 2025-01-01 00:00:00

SELECT DATE_TRUNC('month', TIMESTAMP '2025-02-25 13:54:44');
-- Result: 2025-02-01 00:00:00

7. String Concatenation and Case Functions

String Concatenation

-- Using || Operator
SELECT first_name || ' ' || last_name AS full_name FROM customer;

-- Using CONCAT() Function
SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM customer;

Case Functions

FunctionDescriptionExample
UPPER(string)Converts to uppercaseUPPER('hello') → 'HELLO'
LOWER(string)Converts to lowercaseLOWER('HELLO') → 'hello'
INITCAP(string)Converts to title caseINITCAP('hello world') → 'Hello World'

REPLACE() Function

SELECT REPLACE(title, ' ', '_') AS title FROM film;

8. Parsing String and Character Data

String Length Functions

SELECT CHAR_LENGTH(title) FROM film;
SELECT LENGTH(title) FROM film;

Finding Character Position

SELECT email, POSITION('@' IN email) AS at_position FROM customer;

Extracting Substrings

-- First 50 characters
SELECT LEFT(description, 50) FROM film;

-- Last 50 characters
SELECT RIGHT(description, 50) FROM film;

-- Substring
SELECT SUBSTRING(description, 10, 50) FROM film;

Practical Examples

-- Extract username from email
SELECT LEFT(email, POSITION('@' IN email)-1) AS username FROM customer;

-- Extract domain from email
SELECT SUBSTRING(email FROM POSITION('@' IN email)+1) AS domain FROM customer;

9. Truncating and Padding Strings

TRIM Functions

FunctionDescription
TRIM(string)Removes whitespace from both ends
LTRIM(string)Removes whitespace from left
RTRIM(string)Removes whitespace from right

PAD Functions

SELECT LPAD('padded', 10, '#');  -- Result: '####padded'
SELECT RPAD('padded', 10, '#');  -- Result: 'padded####'

10. Full-Text Search

LIKE Operator

WildcardMeaning
%Zero or more characters
_Exactly one character

Full-Text Search Functions

SELECT title, description
FROM film
WHERE to_tsvector(title) @@ to_tsquery('elf');

Key Advantages over LIKE:

  • Case-insensitive
  • Accounts for word variations (lexemes)
  • Supports stemming and fuzzy matching
  • Better performance

11. PostgreSQL Extensions

Managing Extensions

-- View available extensions
SELECT name FROM pg_available_extensions;

-- View enabled extensions
SELECT extname FROM pg_extension;

-- Enable an extension
CREATE EXTENSION IF NOT EXISTS fuzzystrmatch;

Common Extensions

ExtensionPurpose

Levenshtein Distance

SELECT levenshtein('GUMBO', 'GAMBOL');  -- Result: 2

Trigram Similarity

SELECT similarity('GUMBO', 'GAMBOL');  -- Result: 0.181818

12. User-Defined Types and Functions

Creating ENUM Data Types

CREATE TYPE dayofweek AS ENUM (
    'Monday', 'Tuesday', 'Wednesday', 
    'Thursday', 'Friday', 'Saturday', 'Sunday'
);

Creating User-Defined Functions

CREATE FUNCTION squared(i integer) RETURNS integer AS $$
    BEGIN
        RETURN i * i;
    END;
$$ LANGUAGE plpgsql;

SELECT squared(10);  -- Returns 100
