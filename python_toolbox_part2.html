1. Introduction to Iterators

What is an Iterable?

An iterable is an object that has an associated iter() method. Examples: lists, strings, dictionaries, file connections, range objects.

What is an Iterator?

An iterator is an object that keeps state and produces the next value when you call next() on it.

word = 'hello'

# Create an iterator from an iterable
it = iter(word)

# Get next values
print(next(it))  # 'h'
print(next(it))  # 'e'
print(next(it))  # 'l'
print(next(it))  # 'l'
print(next(it))  # 'o'
print(next(it))  # StopIteration error

Iterating with *

The * operator can unpack all remaining elements.

word = 'data'
it = iter(word)

print(*it)  # d a t a

2. enumerate() and zip()

enumerate()

Returns an enumerate object that produces (index, element) pairs.

names = ['Alice', 'Bob', 'Charlie']

for index, name in enumerate(names):
    print(index, name)
# 0 Alice
# 1 Bob
# 2 Charlie

# Start from a different index
for index, name in enumerate(names, start=1):
    print(index, name)
# 1 Alice
# 2 Bob
# 3 Charlie

zip()

Zips together multiple iterables into an iterator of tuples.

names = ['Alice', 'Bob']
ages = [25, 30]
cities = ['NYC', 'LA']

# Zip together
zipped = zip(names, ages, cities)
print(list(zipped))  # [('Alice', 25, 'NYC'), ('Bob', 30, 'LA')]

# Unzip with *
z = zip(names, ages)
names2, ages2 = zip(*z)
print(names2)  # ('Alice', 'Bob')
print(ages2)   # (25, 30)

3. Processing Large Files in Chunks

Reading with pandas in chunks

import pandas as pd

# Read in chunks
result = []
for chunk in pd.read_csv('data.csv', chunksize=100):
    # Process each chunk
    result.append(sum(chunk['x']))

total = sum(result)
print(total)

Read specific columns only

df = pd.read_csv('data.csv', usecols=['x', 'y'])

4. List Comprehensions

Basic Syntax

# Collapse for loop into a single line
nums = [12, 8, 21, 3, 16]

new_nums = [num + 1 for num in nums]
print(new_nums)  # [13, 9, 22, 4, 17]

Equivalence

# For loop version
new_nums = []
for num in nums:
    new_nums.append(num + 1)

# List comprehension version
new_nums = [num + 1 for num in nums]

Over range()

result = [num ** 2 for num in range(11)]
# [0, 1, 4, 9, 16, 25, 36, 49, 64, 81, 100]

Nested Loops

pairs = [(num1, num2) for num1 in range(0, 2) for num2 in range(6, 8)]
# [(0, 6), (0, 7), (1, 6), (1, 7)]

5. Advanced List Comprehensions

Conditionals in List Comprehensions

# If condition (filtering)
[num ** 2 for num in range(10) if num % 2 == 0]
# [0, 4, 16, 36, 64]

# If-else (transformation)
[num if num % 2 == 0 else 0 for num in range(10)]
# [0, 0, 2, 0, 4, 0, 6, 0, 8, 0]

Note: if alone goes at the end; if-else goes before the for.

Dictionary Comprehensions

pos_neg = {num: -num for num in range(9)}
# {0: 0, 1: -1, 2: -2, 3: -3, ...}

6. Introduction to Generator Expressions

Generator vs List Comprehension

List ComprehensionGenerator Expression
Uses []Uses ()
Returns a listReturns a generator object
Stores all values in memoryProduces values on demand
Good for small datasetsGood for large datasets
# List comprehension - stores in memory
result = [num for num in range(1000000)]

# Generator expression - lazy evaluation
result = (num for num in range(1000000))

Using Generators

gen = (num ** 2 for num in range(10))

# Iterate through
for num in gen:
    print(num)

# Or use next()
print(next(gen))  # 0
print(next(gen))  # 1

Conditionals in Generators

# Even squares
even_squares = (num ** 2 for num in range(10) if num % 2 == 0)
print(list(even_squares))  # [0, 4, 16, 36, 64]

7. Generator Functions

yield vs return

  • return: Returns a value and terminates the function
  • yield: Returns a value but pauses the function, keeping state
def num_sequence(n):
    """Generate values from 0 to n."""
    i = 0
    while i < n:
        yield i
        i += 1

# Create generator
gen = num_sequence(5)

# Iterate
for num in gen:
    print(num)  # 0, 1, 2, 3, 4

Practical Example: Streaming Data

def read_large_file(file_object):
    """Lazily read a file line by line."""
    while True:
        data = file_object.readline()
        if not data:
            break
        yield data

# Usage
with open('huge_file.txt') as file:
    gen = read_large_file(file)
    print(next(gen))  # First line
    print(next(gen))  # Second line

Best Practices

  1. Use generators for large datasets - saves memory
  2. Use list comprehensions for small datasets - more readable
  3. Use enumerate() instead of range(len())
  4. Use zip() to iterate over multiple sequences
  5. Process large files in chunks
  6. Use yield for streaming/lazy evaluation
