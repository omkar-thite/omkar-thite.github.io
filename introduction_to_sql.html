Introduction to SQL

What is SQL?

SQL (Structured Query Language) is used to communicate with relational databases. It allows you to query, manipulate, and manage data stored in tables.

Basic Query Structure

SELECT Statement

The SELECT statement retrieves data from a database table.

SELECT column1, column2
FROM table_name;

Example: Select all columns from a table:

SELECT * 
FROM books;

Selecting Specific Columns

SELECT title 
FROM books;

Selecting Multiple Columns

When selecting multiple columns, separate them with commas:

SELECT title, author, year
FROM books;

SQL Best Practices & Conventions

ConventionDescription
Table namesUse plural names (e.g., books, users)
Column namesUse singular names (e.g., title, author)
KeywordsCapitalize reserved keywords (SELECT, FROM, WHERE)
SemicolonEnd each query with a semicolon (;)

Column Aliases (AS)

Use AS to rename a column in the result set. This doesn't change the actual column name in the database.

SELECT title AS book_title 
FROM books;

DISTINCT Keyword

The DISTINCT keyword removes duplicate values from the result set.

Single Column

SELECT DISTINCT genre
FROM books;

Multiple Columns

When used with multiple columns, DISTINCT returns unique combinations of all specified columns.

SELECT DISTINCT genre, author 
FROM books;

Key Point: Individual values may repeat, but no complete row (combination of all columns) will be duplicated.

LIMIT Clause

The LIMIT clause restricts the number of rows returned.

SELECT genre 
FROM books 
LIMIT 10;

Why Use LIMIT?

  • Testing: Write and test code with a small number of results before running on full dataset
  • Performance: Avoid loading thousands of records when you only need a few
  • Preview: Quickly preview table structure and data

Views

A view is a virtual table created from a saved SQL SELECT statement.

Key Characteristics

FeatureDescription
VirtualData is not stored separately; the query is stored and executed when accessed
Auto-updateAutomatically reflects changes in the underlying database
No result setCreating a view doesn't return any data
ReusableCan be queried like a regular table

Creating a View

CREATE VIEW view_name AS
SELECT DISTINCT author, year, genre
FROM books;

Querying a View

SELECT * FROM view_name;

Benefits of Views

  1. Save complex queries for future reference
  2. Create custom data representations without modifying the database
  3. Useful when you don't have permission to edit the database
  4. Simplify complex queries by breaking them into reusable components

SQL Flavors (Dialects)

SQL has different versions or "flavors." All are used with relational databases and share a universal standard for core functionality.

Tip: Core SQL skills transfer across flavors. The differences are mainly in additional features and minor syntax variations.

Quick Reference Cheat Sheet

-- Select all columns
SELECT * FROM table_name;

-- Select specific columns
SELECT column1, column2 FROM table_name;

-- Alias a column
SELECT column1 AS alias_name FROM table_name;

-- Remove duplicates
SELECT DISTINCT column1 FROM table_name;

-- Limit results
SELECT * FROM table_name LIMIT 10;

-- Create a view
CREATE VIEW view_name AS
SELECT column1, column2 FROM table_name;

-- Query a view
SELECT * FROM view_name;
