Software Engineering for Data Scientists in Python

Best practices for writing maintainable, modular, and well-documented Python code.

1. Software Engineering Concepts

Modularity

Break code into smaller, logical, and reusable modules.

  • Functions: Group related operations
  • Modules: Python files containing related functions
  • Packages: Collections of modules in directories

Documentation

Write clear documentation to explain code purpose and usage.

  • Comments: Inline explanations
  • Docstrings: Function/class documentation
  • README: Project-level documentation

Testing

Verify code works as expected through automated tests.

2. Writing Python Modules

Module Structure

# my_funcs.py

def document_it(func):
    """A decorator that prints function info."""
    def wrapper(*args, **kwargs):
        print(f'Running {func.__name__}')
        return func(*args, **kwargs)
    return wrapper

@document_it
def add(a, b):
    """Add two numbers."""
    return a + b

Importing Modules

# Import entire module
import my_funcs
my_funcs.add(1, 2)

# Import specific function
from my_funcs import add
add(1, 2)

# Import with alias
import my_funcs as mf
mf.add(1, 2)

3. Creating Python Packages

Package Structure

my_package/
├── __init__.py      # Makes it a package
├── module1.py
├── module2.py
└── subpackage/
    ├── __init__.py
    └── module3.py

__init__.py

This file makes a directory a Python package. It can be empty or contain initialization code.

# my_package/__init__.py
from .module1 import function1
from .module2 import function2

__all__ = ['function1', 'function2']

Importing from Packages

from my_package import function1
from my_package.subpackage import module3

4. PEP 8 Style Guide

Naming Conventions

TypeConventionExample

Modulessnake_casemy_module.py

Formatting Rules

  • Indentation: 4 spaces (not tabs)
  • Line length: Max 79 characters
  • Blank lines: 2 between top-level definitions, 1 between methods
  • Imports: One per line, grouped (stdlib, third-party, local)
# Good
import os
import sys

import numpy as np
import pandas as pd

from my_package import my_function

Use triple quotes for function documentation.

def calculate_area(length, width):
    """
    Calculate the area of a rectangle.

6. Classes in Python

Basic Class Structure

class DataProcessor:
    """A class for processing data."""
    
    # Class attribute (shared by all instances)
    version = "1.0"
    
    def __init__(self, data):
        """Initialize with data."""
        self.data = data  # Instance attribute
    
    def process(self):
        """Process the data."""
        return [x * 2 for x in self.data]
    
    def __repr__(self):
        """String representation."""
        return f"DataProcessor(data={self.data})"

Using the Class

processor = DataProcessor([1, 2, 3])
print(processor.process())  # [2, 4, 6]
print(processor.version)    # 1.0
print(processor)            # DataProcessor(data=[1, 2, 3])

Inheritance

class AdvancedProcessor(DataProcessor):
    """Extended processor with additional features."""
    
    def __init__(self, data, multiplier=2):
        super().__init__(data)  # Call parent __init__
        self.multiplier = multiplier
    
    def process(self):
        """Override parent method."""
        return [x * self.multiplier for x in self.data]

7. DRY Principle

Don't Repeat Yourself - avoid code duplication.

Bad Practice

# Repeated code
df1_clean = df1.dropna().reset_index(drop=True)
df2_clean = df2.dropna().reset_index(drop=True)
df3_clean = df3.dropna().reset_index(drop=True)

Good Practice

def clean_dataframe(df):
    """Clean a dataframe."""
    return df.dropna().reset_index(drop=True)

df1_clean = clean_dataframe(df1)
df2_clean = clean_dataframe(df2)
df3_clean = clean_dataframe(df3)

8. Unit Testing with pytest

Basic Test

# test_functions.py
from my_module import add

def test_add_positive():
    assert add(2, 3) == 5

def test_add_negative():
    assert add(-1, -1) == -2

def test_add_zero():
    assert add(0, 0) == 0

Running Tests

# Run all tests
pytest

# Run specific file
pytest test_functions.py

# Run with verbose output
pytest -v

Testing for Exceptions

import pytest

def test_divide_by_zero():
    with pytest.raises(ZeroDivisionError):
        1 / 0

Best Practices Summary

  1. Follow PEP 8 for consistent style
  2. Write docstrings for all functions and classes
  3. Use meaningful names for variables and functions
  4. Keep functions small and focused on one task
  5. Write tests for critical functionality
  6. Don't repeat yourself (DRY)
  7. Use version control (Git)
  8. Document your code with README and comments
