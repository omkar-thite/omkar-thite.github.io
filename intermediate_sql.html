Intermediate SQL - Complete Notes

A comprehensive guide covering COUNT, filtering, aggregate functions, sorting, grouping, and more.

1. COUNT and DISTINCT

COUNT()

Returns the number of records with values in a field.

-- Count non-null values in a specific field
SELECT COUNT(birthdate) AS count_birthdates
FROM people;

-- Count ALL records in a table (including nulls)
SELECT COUNT(*) AS total_records
FROM people;
FunctionDescription
COUNT(field_name)Counts non-missing values in that field
COUNT(*)Counts all records including rows with NULL values

DISTINCT

Removes duplicates from results.

-- Get unique languages
SELECT DISTINCT language
FROM films;

Combining COUNT and DISTINCT

SELECT COUNT(DISTINCT birthdate) AS count_distinct_birthdates
FROM people;

3. Debugging and SQL Style

Best Practices

-- Good formatting
SELECT 
    field_a, 
    field_b,
    field_c
FROM table_name
WHERE condition = value;
  • Capitalize keywords: SELECT, FROM, WHERE, etc.
  • Add newlines between clauses
  • Indent field names for multiple SELECT fields
  • Use semicolons at end of queries
  • Avoid spaces in field names: use release_year not release year

Filtering Strings

Use single quotes around string values:

5. Multiple Criteria: AND, OR, BETWEEN

OR Operator

Returns record if at least one condition is satisfied.

-- ✅ Valid: specify field for each condition
SELECT title
FROM movies
WHERE year = 1960 OR year = 1972;

AND Operator

All conditions must be satisfied.

SELECT title
FROM movies
WHERE year > 1994 AND year < 2000;

Combining AND & OR

Use parentheses to ensure correct execution order:

SELECT title 
FROM movies 
WHERE (year = 1994 OR year = 1995)
    AND (certified = 'PG' OR certified = 'R');

BETWEEN Operator

Filter values within a range (inclusive of both ends).

-- These are equivalent
SELECT title FROM movies WHERE year >= 1994 AND year <= 2000;
SELECT title FROM movies WHERE year BETWEEN 1994 AND 2000;

Remember: BETWEEN is inclusive - includes both beginning and end values.

6. Filtering Text: LIKE, NOT LIKE, IN

LIKE with Wildcards

WildcardMeaning
%Matches zero, one, or more characters
_Matches exactly one character
-- Names starting with 'Ade' (Adele, Adelaide, Aden...)
SELECT name FROM people WHERE name LIKE 'Ade%';

-- Exactly 3-letter names starting with 'Ev' (Eve, Eva...)
SELECT name FROM people WHERE name LIKE 'Ev_';

-- Names ending with 'r'
SELECT name FROM people WHERE name LIKE '%r';

IN Operator

Specify multiple values - cleaner than multiple OR conditions.

-- Instead of multiple ORs
SELECT title FROM films WHERE year IN (1920, 1930, 1960);

-- With strings
SELECT title FROM films WHERE country IN ('India', 'Germany');

7. NULL Values

NULL represents missing or unknown values.

-- Find records with missing birthdates
SELECT name FROM people WHERE birthdate IS NULL;

-- Find records with non-null values
SELECT COUNT(*) AS count_birthdates
FROM people WHERE birthdate IS NOT NULL;

Key Point: Cannot use = NULL (must use IS NULL)

8. Aggregate Functions

FunctionDescriptionData Types
COUNT()Count of non-null valuesAll types
AVG()AverageNumeric only

MAX()Maximum valueAll types
SELECT AVG(budget) FROM films;      -- Average budget
SELECT SUM(budget) FROM films;      -- Total budget
SELECT MIN(budget) FROM films;      -- Lowest budget
SELECT MAX(budget) FROM films;      -- Highest budget

Best Practice: Always use aliases with aggregate functions.

Integer Division Gotcha

SELECT (4 / 3);      -- Returns 1 (integer division)
SELECT (4.0 / 3.0);  -- Returns 1.3333... (decimal division)

Tip: Use decimals (e.g., 60.0) when you need decimal results.

-- Calculate profit for each film
SELECT (gross - budget) AS profit FROM films;

-- Duration in hours
SELECT title, ROUND((duration / 60.0), 2) AS duration_hours
FROM films;

11. Sorting with ORDER BY

-- Ascending order (default): smallest to largest, A to Z
SELECT title, budget FROM films ORDER BY budget;
SELECT title, budget FROM films ORDER BY budget ASC;

-- Descending order: largest to smallest, Z to A
SELECT title, budget FROM films ORDER BY budget DESC;

Sorting by Multiple Fields

-- Sort by wins (desc), then by imdb_score (desc) as tie-breaker
SELECT title, wins, imdb_score
FROM films
ORDER BY wins DESC, imdb_score DESC;

12. Grouping with GROUP BY

SELECT certification, COUNT(title) AS title_count
FROM films
GROUP BY certification;

Rules for GROUP BY

  1. All non-aggregated fields in SELECT must be in GROUP BY
  2. GROUP BY creates groups, not ordering
-- ✅ Correct: include all non-aggregated fields
SELECT country, release_year, COUNT(*) AS film_count
FROM films
GROUP BY country, release_year;

Execution Order: FROM → GROUP BY → SELECT → ORDER BY → LIMIT

13. Filtering Groups with HAVING

Cannot use WHERE with aggregate functions. Use HAVING instead.

-- ✅ Valid: Use HAVING for aggregate filtering
SELECT release_year, COUNT(title) AS title_count
FROM films
GROUP BY release_year
HAVING COUNT(title) > 10;

WHERE vs HAVING

ClauseFiltersWhen Executed
WHEREIndividual recordsBefore GROUP BY
HAVINGGrouped recordsAfter GROUP BY

Note: Cannot use alias in HAVING (executes before SELECT)

14. Complete Query Structure

Full Order of Execution

FROM        → Identify table
WHERE       → Filter individual rows
GROUP BY    → Group rows
HAVING      → Filter groups
SELECT      → Select/calculate columns
ORDER BY    → Sort results
LIMIT       → Limit output

Complete Example

SELECT 
    release_year, 
    AVG(budget) AS avg_budget, 
    AVG(gross) AS avg_gross
FROM films
WHERE release_year > 1990           -- Filter rows first
GROUP BY release_year               -- Group by year
HAVING AVG(budget) > 60000000       -- Filter groups
ORDER BY avg_gross DESC             -- Sort results
LIMIT 1;                            -- Get top result

Key Takeaways

  1. COUNT(*) counts all rows; COUNT(field) counts non-null values
  2. DISTINCT removes duplicates
  3. WHERE filters rows; HAVING filters groups
  4. BETWEEN is inclusive of both endpoints
  5. Use parentheses when combining AND and OR
  6. NULL values require IS NULL / IS NOT NULL
  7. Integer division returns integers; use decimals for decimal results
  8. Aliases cannot be used in WHERE (execution order)
  9. ORDER BY comes after GROUP BY
  10. Always alias aggregate functions for clarity
