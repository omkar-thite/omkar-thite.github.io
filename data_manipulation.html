Data Manipulation in SQL - Revision Notes

Comprehensive notes for exam preparation covering conditional logic, subqueries, CTEs, and window functions.

1. Conditional Logic with CASE

Core Concept

CASE statements bring if-then-else logic directly into SQL queries. They evaluate conditions sequentially and return a value when a condition is met.

Structure:

CASE 
    WHEN test_condition THEN output_if_true
    WHEN another_test THEN another_output
    ELSE fallback_value 
END AS column_alias

Important Behaviors

Scenario Result
No ELSE clause, no conditions match Returns NULL
ELSE NULL specified Same as above - returns NULL
Multiple conditions match First matching condition wins

Practical Applications

1. Categorizing Data:

Determine match outcomes based on goals:

SELECT match_id,
    CASE 
        WHEN home_goal > away_goal THEN 'Home Victory'
        WHEN home_goal < away_goal THEN 'Away Victory'
        ELSE 'Draw'
    END AS result
FROM soccer.match;

2. Complex Conditions with AND/OR:

Check both team identity and match result:

SELECT date,
    CASE 
        WHEN hometeam_id = 8455 AND home_goal > away_goal THEN 'Team Won at Home'
        WHEN awayteam_id = 8455 AND away_goal > home_goal THEN 'Team Won Away'
        ELSE 'Did Not Win'
    END AS team_performance
FROM soccer.match
WHERE hometeam_id = 8455 OR awayteam_id = 8455;

3. Using CASE as a Filter:

Include the full CASE expression (without alias) in WHERE:

SELECT season, date, home_goal, away_goal
FROM soccer.match
WHERE 
    CASE 
        WHEN hometeam_id = 9857 AND home_goal > away_goal THEN 'Victory'
        WHEN awayteam_id = 9857 AND away_goal > home_goal THEN 'Victory'
    END IS NOT NULL;

Aggregating with CASE

COUNT specific outcomes:

SELECT season,
    COUNT(CASE WHEN home_goal > away_goal THEN 1 END) AS home_victories,
    COUNT(CASE WHEN away_goal > home_goal THEN 1 END) AS away_victories
FROM soccer.match
GROUP BY season;

SUM for weighted counting:

SELECT country,
    SUM(CASE WHEN result = 'win' THEN 1 ELSE 0 END) AS total_wins
FROM results
GROUP BY country;

AVG for percentages: Return 1 for success, 0 for failure

SELECT team_name,
    ROUND(AVG(CASE 
        WHEN goals_for > goals_against THEN 1.0
        WHEN goals_for < goals_against THEN 0.0
    END) * 100, 1) AS win_percentage
FROM team_stats
GROUP BY team_name;

⚠️ Key Point: When using AVG for percentages, specify both the success (1) AND failure (0) conditions explicitly, or NULLs will skew your results.

2. Subqueries

Overview

A subquery is a query embedded within another query. Look for multiple SELECT keywords to identify them.

Placement Options:

  • WHERE clause → Filter by calculated values
  • FROM clause → Create derived tables
  • SELECT clause → Add computed columns

Subqueries in WHERE

Single Value (Scalar):

Filter matches exceeding average goals:

SELECT date, home_goal, away_goal
FROM soccer.match
WHERE (home_goal + away_goal) > 
    (SELECT AVG(home_goal + away_goal) FROM soccer.match);

List of Values:

Find teams that never played at home:

SELECT team_long_name
FROM soccer.team 
WHERE team_api_id NOT IN
    (SELECT DISTINCT hometeam_id FROM soccer.match);

Subqueries in FROM

Creates a temporary derived table. Must always have an alias.

Finding top performers:

SELECT team_name, avg_score
FROM (
    SELECT 
        t.team_long_name AS team_name,
        AVG(m.home_goal) AS avg_score
    FROM soccer.match m
    JOIN soccer.team t ON m.hometeam_id = t.team_api_id
    WHERE m.season = '2014/2015'
    GROUP BY t.team_long_name
) AS team_averages
WHERE avg_score > 2.0
ORDER BY avg_score DESC;

Subqueries in SELECT

Returns a single scalar value added to each row.

Comparing individual values to averages:

SELECT 
    league_name,
    total_goals,
    total_goals - (SELECT AVG(total_goals) FROM league_stats) AS deviation_from_avg
FROM league_stats;

⚠️ Critical: Filters in the main query do NOT automatically apply to subqueries. Apply filters in BOTH locations for accurate results.

3. Correlated Subqueries

What Makes Them Different?

Aspect Simple Subquery Correlated Subquery
Execution Runs once Runs once per outer row
Dependencies Standalone References outer query
Performance Generally faster Can be slower on large datasets

Syntax Pattern

The inner query references a column from the outer query:

SELECT m.date, m.home_goal, m.away_goal
FROM soccer.match AS m
WHERE (m.home_goal + m.away_goal) > 
    (SELECT AVG(sub.home_goal + sub.away_goal)
     FROM soccer.match AS sub
     WHERE sub.country_id = m.country_id);  -- Correlation here

This finds matches where total goals exceed the country-specific average.

Multi-Column Correlation

Match on multiple criteria:

SELECT country_id, season, home_goal, away_goal
FROM soccer.match AS outer_m
WHERE (home_goal + away_goal) = 
    (SELECT MAX(sub.home_goal + sub.away_goal)
     FROM soccer.match AS sub
     WHERE outer_m.country_id = sub.country_id
       AND outer_m.season = sub.season);

4. Nested Subqueries

For multi-step transformations, nest subqueries inside each other:

SELECT 
    month_num,
    monthly_total,
    monthly_total - avg_monthly AS difference
FROM (
    SELECT 
        EXTRACT(MONTH FROM date) AS month_num,
        SUM(home_goal + away_goal) AS monthly_total,
        (SELECT AVG(monthly_sum) 
         FROM (SELECT SUM(home_goal + away_goal) AS monthly_sum
               FROM soccer.match
               GROUP BY EXTRACT(MONTH FROM date)) AS inner_calc
        ) AS avg_monthly
    FROM soccer.match
    GROUP BY EXTRACT(MONTH FROM date)
) AS outer_calc;

💡 Tip: Build nested queries step-by-step. Test each inner layer before adding the next.

5. Common Table Expressions (CTEs)

Benefits Over Subqueries

  • Readability: Defined at the top, making logic clearer
  • Performance: Computed once, stored in memory
  • Reusability: Reference the same CTE multiple times
  • Chainability: Later CTEs can reference earlier ones

Basic Syntax

WITH cte_name AS (
    SELECT columns
    FROM table
    WHERE conditions
)
SELECT * FROM cte_name;

Multiple CTEs

Separate with commas (no comma after the last one):

WITH 
high_scoring AS (
    SELECT country_id, id 
    FROM soccer.match 
    WHERE (home_goal + away_goal) >= 8
),
low_scoring AS (
    SELECT country_id, id 
    FROM soccer.match 
    WHERE (home_goal + away_goal) <= 1
)
SELECT 
    c.name,
    COUNT(h.id) AS exciting_matches,
    COUNT(l.id) AS boring_matches
FROM soccer.country c
LEFT JOIN high_scoring h ON c.id = h.country_id
LEFT JOIN low_scoring l ON c.id = l.country_id
GROUP BY c.name;

CTEs with JOINs Inside

WITH match_details AS (
    SELECT 
        m.id,
        m.date,
        l.name AS league_name,
        m.home_goal,
        m.away_goal
    FROM soccer.match m
    JOIN soccer.league l ON m.country_id = l.country_id
    WHERE m.season = '2014/2015'
)
SELECT league_name, COUNT(*) AS match_count
FROM match_details
WHERE (home_goal + away_goal) > 5
GROUP BY league_name;

6. Window Functions

Core Concept

Window functions compute values across a set of rows related to the current row—without collapsing rows into groups.

Regular Aggregation Window Function
GROUP BY required No grouping needed
One row per group All rows preserved
SUM(col) SUM(col) OVER()

The OVER() Clause

Empty OVER() applies the calculation across all rows:

SELECT 
    match_date,
    goals_scored,
    AVG(goals_scored) OVER() AS overall_average,
    goals_scored - AVG(goals_scored) OVER() AS diff_from_avg
FROM match_results;

RANK() Function

Assigns rankings based on ORDER BY within OVER():

SELECT 
    team_name,
    total_points,
    RANK() OVER(ORDER BY total_points DESC) AS league_position
FROM standings;

Note: RANK() produces ties and gaps. Use DENSE_RANK() for consecutive numbers or ROW_NUMBER() for unique values.

PARTITION BY

Creates separate "windows" for each partition:

SELECT 
    season,
    team_name,
    goals_scored,
    AVG(goals_scored) OVER(PARTITION BY season) AS season_average
FROM team_performance;

Multiple Partition Columns:

SELECT 
    country,
    season,
    team_name,
    goals,
    SUM(goals) OVER(PARTITION BY country, season) AS country_season_total
FROM results;

Sliding Window Frames

Define a moving calculation window relative to current row.

Keywords:

  • UNBOUNDED PRECEDING → Start of partition
  • n PRECEDING → n rows before current
  • CURRENT ROW → Current row
  • n FOLLOWING → n rows after current
  • UNBOUNDED FOLLOWING → End of partition

Cumulative Sum:

SELECT 
    game_date,
    points_earned,
    SUM(points_earned) OVER(
        ORDER BY game_date 
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS cumulative_points
FROM season_results;

3-Game Rolling Average:

SELECT 
    game_date,
    goals,
    AVG(goals) OVER(
        ORDER BY game_date
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS three_game_avg
FROM team_matches;

7. Choosing the Right Approach

Technique Best For Watch Out For
JOINs Combining related tables Cartesian products with missing keys
Simple Subqueries One-time calculations Repeating complex subqueries
Correlated Subqueries Row-by-row comparisons Performance on large tables
CTEs Complex multi-step logic Over-engineering simple queries
Window Functions Running totals, rankings Not available in all databases

8. Complete Example: Putting It All Together

Scenario: Identify Manchester United's defeats in 2014/2015, ranked by goal margin.

WITH home_matches AS (
    SELECT 
        m.id,
        t.team_long_name AS opponent,
        m.home_goal AS mu_goals,
        m.away_goal AS opponent_goals,
        'Home' AS venue
    FROM soccer.match m
    JOIN soccer.team t ON m.awayteam_id = t.team_api_id
    WHERE m.hometeam_id = (SELECT team_api_id FROM soccer.team WHERE team_long_name = 'Manchester United')
      AND m.season = '2014/2015'
      AND m.home_goal < m.away_goal
),
away_matches AS (
    SELECT 
        m.id,
        t.team_long_name AS opponent,
        m.away_goal AS mu_goals,
        m.home_goal AS opponent_goals,
        'Away' AS venue
    FROM soccer.match m
    JOIN soccer.team t ON m.hometeam_id = t.team_api_id
    WHERE m.awayteam_id = (SELECT team_api_id FROM soccer.team WHERE team_long_name = 'Manchester United')
      AND m.season = '2014/2015'
      AND m.away_goal < m.home_goal
),
all_defeats AS (
    SELECT * FROM home_matches
    UNION ALL
    SELECT * FROM away_matches
)
SELECT 
    opponent,
    venue,
    mu_goals,
    opponent_goals,
    (opponent_goals - mu_goals) AS goal_margin,
    RANK() OVER(ORDER BY (opponent_goals - mu_goals) DESC) AS defeat_rank
FROM all_defeats
ORDER BY defeat_rank;
