Python Testing with Pytest and Unittest

A comprehensive guide to testing in Python using pytest and unittest frameworks.

Introduction to Pytest

Basic Concepts

  • Test function names must start with test_ for pytest to recognize them
  • Pytest uses assert statements to verify expected outcomes

Running Tests

# Run tests in a single file
pytest test_file.py

# Run all tests in a directory
pytest test_dir/

# Run tests matching a keyword (use -k flag)
pytest tests.py -k 'squared'

Testing for Exceptions

import pytest

def divide(a, b):
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b

def test_divide_by_zero():
    with pytest.raises(ValueError):
        divide(10, 0)

Test Markers

Skip Marker

@pytest.mark.skip
def test_feature_not_ready():
    assert False  # This won't run

Conditional Skip (skipif)

import sys

@pytest.mark.skipif('sys.version_info < (3, 9)')
def test_requires_python_39():
    assert True

Expected Failure (xfail)

@pytest.mark.xfail
def test_known_bug():
    assert 1 == 2  # xfailed (failed as expected)

Fixtures

Fixtures provide a prepared environment for test execution.

Basic Fixture

import pytest

@pytest.fixture
def sample_data():
    return [0, 2, 4, 6, 7, 8, 9, 5, 1]

def test_list(sample_data):
    assert len(sample_data) == 9
    assert 5 in sample_data

Chaining Fixtures

@pytest.fixture
def list_length():
    return 10

@pytest.fixture
def prepare_list(list_length):
    return [i for i in range(list_length)]

def test_elements(prepare_list):
    assert 9 in prepare_list

Autouse Fixtures

@pytest.fixture(autouse=True)
def set_pd_options():
    pd.set_option('display.max_columns', 5000)

def test_pd_options():
    assert pd.get_option('display.max_columns') == 5000

Fixture Teardowns

@pytest.fixture
def prepare_data():
    # Setup
    data = [i for i in range(10)]
    
    # Yield the data to the test
    yield data
    
    # Teardown (runs after test completes)
    data.clear()
    del data

Types of Testing

1. Unit Testing

Tests the smallest working parts of software (functions, methods, classes).

def sum_of_arr(array: list) -> int:
    return sum(array)

def test_regular():
    assert sum_of_arr([1, 2, 3]) == 6

def test_empty():
    assert sum_of_arr([]) == 0

def test_one_number():
    assert sum_of_arr([10]) == 10

2. Feature Testing

Tests features from the user's perspective.

def filter_data(df, company):
    return df[df['company'] == company]

def test_filter_feature():
    filtered = filter_data(df, 'apple')
    assert filtered['company'].nunique() == 1
    assert filtered['company'].unique() == ['apple']

3. Integration Testing

Verifies that different modules interact correctly.

import os
import pytest

@pytest.fixture
def setup_file():
    with open('file.txt', 'w') as file:
        file.write('test data')
        yield file
    os.remove('file.txt')

def test_file_exists(setup_file):
    assert os.path.exists('file.txt')

4. Performance Testing

Measures efficiency of resource utilization.

# pip install pytest-benchmark

def test_sleep(benchmark):
    benchmark(time.sleep, 1)

Unittest Framework

Key Features

  • Built-in Python framework
  • Object-oriented: Tests are methods within TestCase classes
  • Uses assertion methods instead of assert statements

Basic Structure

import unittest

class TestSquared(unittest.TestCase):
    def test_negative(self):
        self.assertEqual((-3) ** 2, 9)

Assertion Methods

MethodDescription
.assertEqual(a, b)Check a == b
.assertNotEqual(a, b)Check a != b
.assertTrue(x)Check x is True
.assertFalse(x)Check x is False
.assertIs(a, b)Check a is b
.assertIsNone(x)Check x is None
.assertIsInstance(a, type)Check type of object
.assertIn(a, b)Check a in b
.assertRaises(Exception)Check exception is raised

Example

import unittest

def func_factorial(number):
    if number < 0:
        raise ValueError('Factorial not defined for negatives')
    factorial = 1
    while number > 1:
        factorial *= number
        number -= 1
    return factorial

class TestFactorial(unittest.TestCase):
    def test_positives(self):
        self.assertEqual(func_factorial(5), 120)
    
    def test_zero(self):
        self.assertEqual(func_factorial(0), 1)
    
    def test_negatives(self):
        with self.assertRaises(ValueError):
            func_factorial(-2)

Fixtures in Unittest

  • setUp(): Runs before each test method
  • tearDown(): Runs after each test method
class TestList(unittest.TestCase):
    def setUp(self):
        self.li = [i for i in range(100)]
    
    def tearDown(self):
        self.li.clear()
    
    def test_list(self):
        self.assertIn(99, self.li)
        self.assertNotIn(100, self.li)

Best Practices

  1. Define test cases before writing test code
  2. Use fixtures to avoid code duplication
  3. Clean up resources in teardown
  4. Use appropriate test types for different scenarios
  5. Name test functions descriptively
  6. Test edge cases (empty inputs, wrong types, boundary values)
  7. Keep tests independent - each test should be able to run alone
