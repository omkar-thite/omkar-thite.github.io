PostgreSQL Summary Stats and Window Functions

A comprehensive guide to window functions in PostgreSQL for data analysis and reporting.

1. Introduction to Window Functions

What are Window Functions?

Window functions perform calculations across a set of rows that are related to the current row, without collapsing rows like GROUP BY does.

Key Characteristics

  • Similar to aggregate functions but all rows remain in the output
  • Operates on a "window" of rows defined by the OVER() clause
  • Can access values from other rows relative to the current row

Common Use Cases

Use CaseDescription
Fetch previous/next valuesCompare current row with adjacent rows
Running totalsCalculate cumulative sums
Moving averagesSmooth out data trends
RankingAssign ranks based on values
Growth calculationsCompare current vs previous periods

Basic Syntax

FUNCTION_NAME() OVER (
    [PARTITION BY column]
    [ORDER BY column]
    [ROWS/RANGE BETWEEN ... AND ...]
)

2. ROW_NUMBER and ORDER BY

ROW_NUMBER()

Assigns a unique sequential number to each row.

SELECT year, event, country,
    ROW_NUMBER() OVER() AS row_n
FROM summer_medals
WHERE medal = 'Gold';

ORDER BY Inside OVER

SELECT year, event, country,
    ROW_NUMBER() OVER(ORDER BY year DESC) AS row_n
FROM summer_medals
WHERE medal = 'Gold';

Order of Execution

  1. ORDER BY inside OVER() executes first (assigns row numbers)
  2. ORDER BY outside OVER() executes second (sorts final output)

3. PARTITION BY

What is PARTITION BY?

Splits the table into partitions based on column values. Window functions operate separately on each partition.

Key Behaviors

  • ROW_NUMBER() resets to 1 for each partition
  • LAG() only fetches values within the same partition
  • Similar to GROUP BY but rows don't collapse
SELECT 
    year, event, champion,
    LAG(champion, 1) OVER (
        PARTITION BY event 
        ORDER BY year
    ) AS last_champion
FROM discus_gold
ORDER BY event, year;

4. Fetching Functions

FunctionTypeDescription
LAG(col, n)RelativeReturns value n rows before current row
LEAD(col, n)RelativeReturns value n rows after current row
FIRST_VALUE(col)AbsoluteReturns first value in table/partition
LAST_VALUE(col)AbsoluteReturns last value in table/partition

LAG - Previous Row Value

SELECT year, champion,
    LAG(champion, 1) OVER (ORDER BY year ASC) AS last_champion
FROM discus_gold;

LEAD - Next Row Value

SELECT year, city,
    LEAD(city, 1) OVER (ORDER BY year) AS next_city
FROM hosts;

LAST_VALUE - Important Note

⚠️ Important: LAST_VALUE requires special frame syntax!

SELECT year, city,
    LAST_VALUE(city) OVER (
        ORDER BY year 
        RANGE BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS last_city
FROM hosts;

6. Paging with NTILE

NTILE(n) splits data into n approximately equal groups (pages).

Use Cases

  • Pagination for APIs
  • Splitting into quartiles/thirds for analysis
  • Identifying top/middle/bottom performers
SELECT country, medals,
    NTILE(3) OVER (ORDER BY medals DESC) AS third
FROM country_medals;
Third ValueMeaning
1Top 33%
2Middle 33%
3Bottom 33%

7. Aggregate Window Functions

Standard aggregate functions (SUM, MAX, MIN, AVG, COUNT) can be used with OVER().

Running/Cumulative Sum

SELECT year, medals,
    SUM(medals) OVER (ORDER BY year) AS running_total
FROM brazil_medals;

Partitioned Aggregates

SELECT year, country, medals,
    SUM(medals) OVER (
        PARTITION BY country 
        ORDER BY year
    ) AS country_running_total
FROM gold_medals;

8. Frames

What are Frames?

Frames define which rows the window function uses as input.

Frame Boundaries

ClauseMeaning
n PRECEDINGn rows before current row
CURRENT ROWThe current row
n FOLLOWINGn rows after current row
UNBOUNDED PRECEDINGFirst row of partition
UNBOUNDED FOLLOWINGLast row of partition

Frame Examples

Frame DefinitionRows Included
ROWS BETWEEN 3 PRECEDING AND CURRENT ROW4 rows (current + 3 before)
ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING3 rows
ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROWRunning total (default)

9. Moving Averages and Totals

Moving Average

SELECT year, medals,
    AVG(medals) OVER (
        ORDER BY year
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS moving_avg_3yr
FROM usa_golds;

Moving Total

SELECT year, medals,
    SUM(medals) OVER (
        ORDER BY year
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS moving_total_3yr
FROM usa_golds;

10. Pivoting with CROSSTAB

CREATE EXTENSION IF NOT EXISTS tablefunc;

SELECT * FROM CROSSTAB($$
    SELECT country, year, COUNT(*) AS awards
    FROM summer_medals
    WHERE year IN (2008, 2012) AND medal = 'Gold'
    GROUP BY country, year
    ORDER BY country, year
$$) AS ct (
    country VARCHAR,
    "2008" INTEGER,
    "2012" INTEGER
)
ORDER BY country;

11. ROLLUP and CUBE

ROLLUP

Hierarchical aggregation - de-aggregates from left to right.

SELECT country, medal, COUNT(*) AS awards
FROM summer_medals
WHERE year = 2008
GROUP BY ROLLUP(country, medal)
ORDER BY country, medal;

CUBE

Non-hierarchical - generates all possible group-level aggregations.

SELECT gender, medal, COUNT(*) AS awards
FROM summer_medals
WHERE year = 2012 AND country = 'RUS'
GROUP BY CUBE(gender, medal);
ClauseGenerates
ROLLUP(a, b)(a,b), (a), ()
CUBE(a, b)(a,b), (a), (b), ()

12. Useful Functions

COALESCE - Replace NULLs

Returns the first non-null value from left to right.

SELECT
    COALESCE(country, 'All countries') AS country,
    COALESCE(medal, 'All medals') AS medal,
    COUNT(*) AS awards
FROM summer_medals
WHERE year = 2008
GROUP BY ROLLUP(country, medal);

STRING_AGG - Concatenate Values

SELECT STRING_AGG(country, ', ')
FROM country_ranks
WHERE rank <= 3;
-- Result: "USA, RUS, AUS"

Tips for Exams

  1. Window vs Aggregate: Window functions keep all rows; aggregates collapse them
  2. PARTITION BY: Like GROUP BY but doesn't collapse rows
  3. RANK vs DENSE_RANK: RANK skips numbers after ties, DENSE_RANK doesn't
  4. LAST_VALUE needs a frame: Use RANGE BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
  5. ROLLUP is hierarchical: Order of columns matters
  6. COALESCE for NULLs: First non-null value from list
