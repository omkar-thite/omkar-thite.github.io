Joining Data in SQL - Complete Notes

Comprehensive revision notes for SQL JOINs, Set Operations, and Subqueries

1. INNER JOIN

What is a JOIN?

  • JOINs enable combining data from separate tables into meaningful result sets
  • A key is a single column or group of columns that uniquely identifies records in a table
  • You can JOIN on key fields or any other field

How INNER JOIN Works

  • Returns only records that match on the joining field in both tables
  • Records without matches in either table are discarded
  • Matched records are combined into a single row with columns from both tables

Basic Syntax

SELECT 
    table1.column,
    table2.column
FROM table1
INNER JOIN table2
ON table1.key = table2.key;

Key Points

ConceptDescription
Ambiguous columnsWhen a column exists in both tables, use table.column format
Table aliasesUse AS keyword to shorten table names
USING keywordWhen joining on identical column names: USING(column_name)

Examples

-- Basic INNER JOIN
SELECT 
    presidents.country,
    prime_minister,
    president
FROM presidents
INNER JOIN prime_ministers
ON presidents.country = prime_ministers.country;

-- With Aliases
SELECT p1.country, president, prime_minister
FROM presidents AS p1
INNER JOIN prime_ministers AS p2
ON p1.country = p2.country;

-- Using USING Keyword
SELECT p1.country, president, prime_minister
FROM presidents p1
INNER JOIN prime_ministers AS p2
USING(country);

Multiple JOINs

SELECT *
FROM left_table
INNER JOIN right_table ON left_table.id = right_table.id
INNER JOIN another_table ON left_table.id = another_table.id;

Joining on Multiple Conditions

SELECT * 
FROM left_table
INNER JOIN right_table
ON left_table.id = right_table.id
    AND left_table.date = right_table.date;

2. Outer JOINs

LEFT JOIN

  • Returns all records from the left table
  • Returns matching records from the right table
  • Assigns NULL for right table columns where no match exists
SELECT p1.country, prime_minister, president
FROM prime_ministers AS p1
LEFT JOIN presidents AS p2
USING(country);

RIGHT JOIN

  • Returns all records from the right table
  • Returns matching records from the left table
  • Assigns NULL for left table columns where no match exists

FULL JOIN

  • Combines LEFT JOIN and RIGHT JOIN
  • Returns all records from both tables
  • Assigns NULL where matches aren't found on either side

3. CROSS JOIN

  • Creates all possible combinations of records from two tables
  • Also called a Cartesian product
  • Does NOT require ON or USING clause
SELECT id1, id2
FROM table_1
CROSS JOIN table_2;

💡 Use case: Explore all possible combinations, such as historical language possibilities between countries.

4. Self JOIN

  • Table joined with itself
  • Used to compare values within the same table
  • Aliasing is required
-- Find Country Pairs in Same Continent
SELECT
    p1.country AS country1,
    p2.country AS country2,
    p1.continent	
FROM prime_ministers AS p1
INNER JOIN prime_ministers AS p2
ON p1.continent = p2.continent
    AND p1.country <> p2.country;  -- Exclude self-pairs

5. Set Operations

Set operations stack records on top of each other (unlike JOINs which add columns side by side).

Requirements

RequirementDescription
Same number of columnsBoth queries must SELECT the same number of columns
Matching data typesCorresponding columns must have compatible data types
Field namesResult uses field names from the first query

UNION

Combines records from two tables. Removes duplicates.

SELECT monarch AS leader, country FROM monarchs
UNION
SELECT prime_minister, country FROM prime_ministers;

UNION ALL

Combines records from two tables. Keeps duplicates.

INTERSECT

Returns only records that appear in both tables.

SELECT country FROM prime_ministers
INTERSECT
SELECT country FROM presidents;

EXCEPT

Returns records from the first table that are NOT in the second table.

SELECT monarch, country FROM monarchs
EXCEPT
SELECT prime_minister, country FROM prime_ministers;

Set Operations Summary

OperationReturnsDuplicates

UNION ALLAll records from bothKept
INTERSECTCommon records onlyRemoved

6. Semi JOINs and Anti JOINs

These are not additive - they don't add columns, only filter rows.

Semi JOIN

Returns records from the first table where a condition IS met in the second table.

SELECT country, continent, president
FROM presidents 
WHERE country IN 
    (SELECT country FROM states WHERE indep_year < 1800);

Anti JOIN

Returns records from the first table where a condition is NOT met in the second table.

SELECT country, continent, president
FROM presidents
WHERE continent LIKE '%America'
    AND country NOT IN 
        (SELECT name FROM states WHERE indep_year < 1800);

7. Subqueries

Queries nested inside other queries.

Subqueries in WHERE

SELECT *
FROM populations
WHERE year = 2015
    AND life_expectancy > 1.15 *
        (SELECT AVG(life_expectancy) FROM populations WHERE year = 2015);

Subqueries in SELECT

⚠️ Requires an alias!

SELECT
    DISTINCT continent,
    (SELECT COUNT(*) FROM monarchs
     WHERE monarchs.continent = states.continent) AS monarch_count
FROM states;

Subqueries in FROM

SELECT local_name, sub.lang_num
FROM countries, 
    (SELECT code, COUNT(*) AS lang_num
     FROM languages GROUP BY code) AS sub
WHERE countries.code = sub.code
ORDER BY lang_num DESC;
